# Roles

This folder contains durable Geistr rules and architectural roles.

Each role file should explain how one important part of the app works and what constraints future work must follow.

Planned examples:

- `skills.md` — how Geistr thinks about Pi skills and user-created skills.
- `apps.md` — how Geistr models apps/workspaces/projects.
- `loops.md` — how Geistr uses reusable controlled loop workflows for background/supervised agent work.
- `code-agent.md` — how the coding agent behaves and what responsibilities belong to Pi SDK vs Geistr UI.
- `design.md` — UI/product design rules for the desktop app.
